技术写作能力模型

资源推荐​

这两天阅读了两篇文章，有点子小感悟，结合以前的一些碎片想法输出了一份技术写作能力模型。

首先，祭出出处，顺便推荐这俩公众号，个人感觉文章质量是比较稳定的高，时常能触发自己作出一些思考。

  • AI时代最重要的技能

    说明：六镇发表的文章。大V的备用号，有关注的应该很快能品出他的风格。

  • 行业这么“卷” 为何我们依然热爱？

    说明：舜禹环球通Transphere发表的文章。作为舜禹的兼职译员合作过，但是后来因为工作时间原因没怎么接稿了。以前做过翻译行业，舜禹毋庸置疑是很专业的那档。PS：个人感觉写作翻译不分家，虽然需要的技能有差异，但是关联性很强。

其次，叠甲，本人只是一名技术写作从业者，但是半路出家非科班出身，发表的一些看法是基于个人工作经历的一些思考。如有不符合行家认知的，欢迎指正，联系方式请见首页页脚 > More > 有意来聊。

能力模型​

先贴上技术写作能力模型的简图。

思路大致是这样：

  1. 基础技能是敲门砖，从事文字相关的工作，一些写作、翻译技能是必须的，而从事技术类写作，有些编码基础也合理吧，另外技术写作经常需要了解新的技术、和产研等其他职能沟通、验证一些操作，那么搜索解惑、和人沟通、使用主流工具这些也是不可或缺的软实力。
  2. 能力扩展是上升的阶梯，估计从事技术写作的人多多少少都遇到过一个尴尬的情况，费劲巴拉的磨出一篇文档，发出review后这里不行那里不行，技术和编辑评审意见一大堆，自己却不理解这些人为什么吹毛求疵，本着向内求索的原则，还是要提升自己对不同领域的技术、不同岗位的职能和业界公认标准的学习，让自己有综合多方面因素完成架构和内容设计的能力，即使还是遇到质疑，回怼时也能做到有理有据不是？
  3. 其实做到前两步应该已经能hold住大部分的日常写作任务了，但是不出意外的话，内卷是早晚会来的，也见过不少人分享横向突破和纵向突破的方向，但是个人认为，方向一直会有新的，拥有快速适应新方向的能力才是杀手锏，而这种能力不是一蹴而就的。打造个人的知识体系是锻造这种能力的基石，清晰的认知框架、可落地方法论、全面的组织视野则是必不可少的内功。或者尝试反问自己，在能hold住日常写作任务后，自己要不断加强哪些方面才会有能力为企业从0到1打造适合的文档系统。
  4. 虽然简单区分了三个层级的能力，但也要时刻提醒自己，不是有了点心得的后就可以高枕无忧的。时代在发展、市场在洗牌、技术在进步，无论任何职能，都要做到面向最终的业务场景不断输出创作和解决问题，不断迭代自己的能力、扩展自己的视野、更新自己的体系。毕竟，知道的越多，越能发现自己不知道的越多。

写在最后​

可能有朋友要说了：我就想简简单单写个文档，搞这么多花花的干嘛？

这么说也没毛病，还是看自身的需要，在技术写作这个领域当做跳板还是想要深耕。

于我而言，我想知道一篇文档为什么需要写，想知道写了能用在哪里，发挥了什么作用，是否解决了什么问题，而这才是TW的价值所在。

如果你也在苦恼，TW工作时不受重视，沟通难，工作推不动，应该多少能理解这种心情，只有全面的知其所以然，才能知所进退，毕竟任何职能最终都要服务于战略、服务于业务、服务于市场，能做到多少是一回事，去不去尝试是另一回事。
